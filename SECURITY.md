# Security Policy

## Supported Versions

This repository is currently maintained on the latest `main` branch only.

## Reporting a Vulnerability

Please do not open public issues for sensitive security reports.

Preferred reporting path:

1. Open a private GitHub security advisory if the repository has advisories enabled.
2. If that is unavailable, contact the repository owner privately through the GitHub account listed on the repository.

When reporting, include:

- affected component or file path
- reproduction steps
- impact assessment
- any suggested fix or mitigation

I will acknowledge valid reports as quickly as possible and prioritize fixes for credential exposure, auth bypass, injection, and unsafe file or network handling.
